# Web3 Wallet - 註冊/登錄流程原型

這是一個完整的註冊/登錄流程可交互原型，根據 Figma 設計稿實現。

## 文件說明

- `app.html` - 多頁面應用主文件
- `app.js` - 頁面路由和交互邏輯
- `styles.css` - 樣式文件（與單頁版本共享）

## 頁面流程

### 1. 歡迎頁面 (Welcome)
- **路徑**: `page-welcome`
- **功能**: 
  - 創建賬戶按鈕
  - 登錄鏈接
- **交互**: 點擊按鈕進入對應流程

### 2. 輸入手機號 - 註冊 (Phone Signup)
- **路徑**: `page-phone-signup`
- **功能**:
  - 國家代碼選擇器（點擊可選擇國家）
  - 手機號輸入（自動格式化）
  - 協議複選框
  - 創建賬戶按鈕
- **驗證**: 
  - 手機號必須至少 11 位數字
  - 必須同意協議才能創建賬戶
- **流程**: 
  - 成功 → 驗證碼頁面
  - 賬戶已存在 → 顯示模態框提示登錄

### 3. 輸入手機號 - 登錄 (Phone Login)
- **路徑**: `page-phone-login`
- **功能**:
  - 國家代碼選擇器
  - 手機號輸入
  - 登錄按鈕
- **驗證**: 手機號必須至少 11 位數字
- **流程**: 成功 → 驗證碼頁面

### 4. 國家選擇 (Country Select)
- **路徑**: `page-country-select`
- **功能**:
  - 搜索框（實時搜索國家）
  - 國家列表（顯示國旗、國家名、代碼）
  - 空狀態提示
- **交互**: 
  - 點擊國家項更新國家代碼
  - 自動返回上一頁

### 5. 輸入驗證碼 (Verify Code)
- **路徑**: `page-verify-code`
- **功能**:
  - 6 位驗證碼輸入框（自動跳轉）
  - 重新發送鏈接（驗證失敗時顯示）
  - 驗證按鈕
- **交互**:
  - 支持粘貼驗證碼
  - 支持退格鍵刪除
  - 自動聚焦下一個輸入框
- **流程**: 
  - 成功 → 註冊成功頁面
  - 失敗 → 顯示重新發送選項

### 6. 協議頁面 (Terms)
- **路徑**: `page-terms`
- **功能**: 顯示隱私政策和服務條款
- **訪問**: 從註冊頁面的協議鏈接進入

### 7. 註冊成功 (Success)
- **路徑**: `page-success`
- **功能**: 
  - 成功提示
  - 返回首頁按鈕
- **流程**: 點擊按鈕返回歡迎頁面

## 使用方法

### 直接在瀏覽器中打開

1. 雙擊 `app.html` 文件
2. 或在瀏覽器中打開該文件

### 使用本地服務器（推薦）

```bash
# 使用 Python
python -m http.server 8000

# 或使用 Node.js
npx http-server

# 然後訪問
# http://localhost:8000/app.html
```

## 流程測試

### 註冊流程
1. 點擊 "Create account"
2. 輸入手機號（至少 11 位）
3. 勾選協議
4. 點擊 "Create account"
5. 輸入驗證碼（6 位）
6. 完成註冊

### 登錄流程
1. 點擊 "Log in"
2. 輸入手機號
3. 點擊 "Log in"
4. 輸入驗證碼
5. 完成登錄

### 國家選擇
1. 在手機號輸入頁面點擊國家代碼
2. 搜索或選擇國家
3. 自動返回並更新代碼

## 狀態管理

應用使用 `AppState` 對象管理全局狀態：

```javascript
{
  currentPage: 'welcome',      // 當前頁面
  phoneNumber: '',             // 手機號
  countryCode: '+086',         // 國家代碼
  countryFlag: '🇨🇳',          // 國旗
  isSignup: true,              // 是否註冊流程
  verificationCode: '',         // 驗證碼
  agreementAccepted: false     // 是否同意協議
}
```

## 注意事項

1. **模擬 API**: 所有 API 調用都是模擬的，使用隨機結果
2. **驗證碼**: 輸入任意 6 位數字即可（有 70% 成功率）
3. **賬戶檢查**: 創建賬戶時有 50% 概率顯示"賬戶已存在"
4. **狀態持久化**: 頁面刷新會重置所有狀態
