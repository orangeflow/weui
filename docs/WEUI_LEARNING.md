# MimicUI 设计与前端规则学习指南

## 什么是 MimicUI？

MimicUI 是由微信官方设计团队为微信 Web 开发量身打造的基础样式库，旨在提供与微信原生视觉体验一致的界面组件。

## 核心设计原则

### 1. 简洁美观
- **极简主义设计理念**：去除冗余元素，保持界面整洁
- **扁平化设计风格**：避免过度装饰，注重内容本身
- **视觉层次清晰**：通过间距、颜色、字体大小建立清晰的视觉层次

### 2. 用户体验优先
- **人性化设计**：符合用户操作习惯
- **操作便捷**：减少用户操作步骤，提高使用效率
- **反馈及时**：提供清晰的操作反馈和状态提示

### 3. 一致性
- **视觉一致性**：与微信原生界面保持一致
- **交互一致性**：遵循微信的交互规范
- **组件一致性**：相同功能的组件在不同场景下保持一致

## 设计规范要点

### 颜色系统
- **主色调**：通常使用微信绿色 (#07C160) 作为主要操作色
- **文本颜色**：
  - 主要文本：深色（#000000 或接近黑色）
  - 次要文本：灰色（#888888 或 #999999）
  - 辅助文本：浅灰色（#BBBBBB）
- **背景色**：
  - 主背景：白色或浅灰色
  - 次要背景：根据层级使用不同灰度

### 字体规范
- **字体大小**：
  - 标题：通常 18-20px（移动端）
  - 正文：14-16px
  - 辅助文字：12-14px
- **字重**：
  - 标题：600-700（Semi-bold/Bold）
  - 正文：400-500（Regular/Medium）
  - 次要文字：400（Regular）

### 间距系统
- **基础间距单位**：通常使用 4px 或 8px 的倍数
- **常用间距**：
  - 小间距：8px、12px
  - 中间距：16px、24px
  - 大间距：32px、48px

### 圆角规范
- **小圆角**：4px（按钮、输入框）
- **中圆角**：8px（卡片、对话框）
- **大圆角**：12px（特殊场景）

## 前端实现规则

### 1. 组件化开发
```html
<!-- 按钮组件示例 -->
<button class="mimicui-btn mimicui-btn_primary">主要按钮</button>
<button class="mimicui-btn mimicui-btn_default">次要按钮</button>
```

### 2. BEM 命名规范
MimicUI 使用 BEM（Block Element Modifier）命名规范：
- **Block（块）**：`.mimicui-btn`
- **Element（元素）**：`.mimicui-btn__icon`
- **Modifier（修饰符）**：`.mimicui-btn_primary`

### 3. 响应式设计
- **移动优先**：优先考虑移动端体验
- **适配不同屏幕**：使用相对单位和媒体查询
- **触摸友好**：按钮和可点击区域至少 44px × 44px

### 4. 交互反馈
- **点击反馈**：使用 `:active` 伪类提供视觉反馈
- **加载状态**：提供加载动画和状态提示
- **错误处理**：清晰的错误提示和引导

## 常用组件

### 1. 按钮（Button）
- 主要按钮：用于主要操作
- 次要按钮：用于次要操作
- 文字按钮：用于辅助操作

### 2. 表单（Form）
- 输入框：清晰的标签和占位符
- 选择器：易于操作的选项
- 复选框/单选框：明确的选中状态

### 3. 列表（List）
- 基础列表：清晰的信息展示
- 带图标的列表：增强视觉识别
- 可点击列表：明确的交互反馈

### 4. 对话框（Dialog）
- 提示对话框：简洁的提示信息
- 确认对话框：明确的操作选项
- 输入对话框：清晰的输入引导

### 5. 提示（Toast）
- 成功提示：绿色图标
- 错误提示：红色图标
- 加载提示：旋转动画

## 应用到当前项目

基于 MimicUI 的设计原则，可以优化当前项目的：

1. **按钮设计**
   - 统一按钮样式和尺寸
   - 添加明确的交互反馈
   - 使用合适的颜色和圆角

2. **表单设计**
   - 清晰的标签和占位符
   - 统一的输入框样式
   - 明确的错误提示

3. **间距系统**
   - 使用 4px/8px 的倍数作为间距单位
   - 保持一致的间距规范

4. **颜色系统**
   - 建立统一的颜色变量
   - 使用语义化的颜色命名

5. **交互反馈**
   - 添加点击反馈效果
   - 提供加载状态提示
   - 优化错误处理体验

## 关键要点总结

1. **简洁至上**：去除不必要的装饰，专注于内容
2. **用户优先**：一切设计决策以用户体验为准
3. **保持一致**：在整个应用中保持视觉和交互的一致性
4. **组件复用**：通过组件化提高开发效率和一致性
5. **响应式设计**：确保在不同设备上都有良好的体验
